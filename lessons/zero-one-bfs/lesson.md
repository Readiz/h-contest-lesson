# 0-1 BFS

0-1 BFS는 간선 비용이 `0` 또는 `1`인 그래프에서 최단거리를 구하는 알고리즘입니다. 일반 BFS처럼 큐를 쓰지만, 비용이 0인 이동은 앞에 넣고 비용이 1인 이동은 뒤에 넣기 위해 `deque`를 사용합니다.

## 0. 선수 지식과 이어지는 레슨

- 선수 지식: BFS, 최단거리 배열, deque
- 함께 보면 좋은 레슨: BFS/DFS와 격자 탐색
- 다음에 볼 레슨: Dijkstra 최단거리

## 1. 간선 비용이 0 또는 1일 때

일반 BFS는 모든 간선 비용이 1일 때만 최단거리를 보장합니다. 비용이 0인 간선이 섞이면, 한 번 이동했는데 거리 증가가 없을 수 있습니다.

```text
u --0--> v  거리 증가 없음
u --1--> w  거리 1 증가
```

이때 Dijkstra를 써도 됩니다. 하지만 비용이 0과 1뿐이라면 우선순위 큐 대신 deque만으로 더 간단하게 처리할 수 있습니다.

## 2. deque를 쓰는 이유

현재 정점 `u`에서 이웃 `v`로 가는 비용이 `0`이면 `dist[v]`는 `dist[u]`와 같습니다. 이 정점은 지금 처리 중인 거리 그룹과 같은 우선순위이므로 deque 앞쪽에 넣습니다.

비용이 `1`이면 다음 거리 그룹이므로 뒤쪽에 넣습니다.

```text
cost 0: push_front
cost 1: push_back
```

이렇게 하면 deque의 앞쪽에는 항상 현재까지 가장 작은 거리 후보가 옵니다.

## 3. 기본 구현

```cpp
struct Edge {
    int to;
    int cost; // 0 or 1
};

vector<int> zeroOneBfs(const vector<vector<Edge>>& graph, int start) {
    const int INF = 1e9;
    int n = (int)graph.size();
    vector<int> dist(n, INF);
    deque<int> dq;

    dist[start] = 0;
    dq.push_back(start);

    while (!dq.empty()) {
        int u = dq.front();
        dq.pop_front();

        for (const Edge& e : graph[u]) {
            int nd = dist[u] + e.cost;
            if (nd >= dist[e.to]) continue;
            dist[e.to] = nd;
            if (e.cost == 0) dq.push_front(e.to);
            else dq.push_back(e.to);
        }
    }

    return dist;
}
```

같은 정점이 deque에 여러 번 들어갈 수 있습니다. 그래도 더 짧은 거리로 갱신될 때만 넣기 때문에 전체 복잡도는 `O(V + E)`입니다.

## 4. Dijkstra와의 관계

0-1 BFS는 특수한 Dijkstra로 볼 수 있습니다.

| 간선 비용 | 대표 알고리즘 |
| --- | --- |
| 모두 1 | BFS |
| 0 또는 1 | 0-1 BFS |
| 음수 없음 | Dijkstra |
| 음수 가능 | Bellman-Ford, SPFA 검토 |

Dijkstra는 우선순위 큐로 가장 작은 거리 후보를 꺼냅니다. 0-1 BFS는 가능한 거리 증가가 0 또는 1뿐이라는 점을 이용해 deque로 같은 효과를 냅니다.

## 5. 격자 상태 그래프 예시

방향을 바꾸는 비용이 1이고, 같은 방향으로 가는 비용이 0인 격자 문제를 생각해 봅시다. 상태는 `(r, c, dir)`이고, 이동 비용은 방향이 바뀌는지에 따라 정합니다.

```cpp
struct State {
    int r;
    int c;
    int dir;
};
```

이런 문제는 단순 칸 방문 배열로는 부족합니다. 같은 칸이라도 어떤 방향으로 들어왔는지에 따라 다음 비용이 달라지므로 방향까지 상태에 포함해야 합니다.

## 6. 시간 복잡도

| 작업 | 시간 |
| --- | --- |
| 초기화 | `O(V)` |
| 간선 relax 전체 | `O(E)` |
| deque 삽입/삭제 | `O(V + E)` |
| 전체 | `O(V + E)` |
| 메모리 | `O(V + E)` |

## 7. 자주 하는 실수

| 실수 | 결과 | 점검 |
| --- | --- | --- |
| 비용 0 간선도 `push_back` | 불필요하게 늦게 처리 | 비용 0은 `push_front` |
| 비용이 2 이상인데 사용 | 최단거리 보장 깨짐 | 간선 비용이 0/1인지 확인 |
| 상태를 너무 작게 잡음 | 같은 칸 재방문 오답 | 방향, 사용한 기회 등 포함 |
| visited를 처음 방문 때 고정 | 더 짧은 경로 차단 | `dist` 갱신 기준으로 처리 |

## 8. 문제를 볼 때 체크할 조건

1. 모든 간선 비용이 정확히 0 또는 1인가?
2. 상태가 정점 하나로 충분한가, 추가 차원이 필요한가?
3. 시작점이 여러 개라면 모두 거리 0으로 deque에 넣어야 하는가?
4. 비용 0 이동이 사이클을 만들 수 있는가?
5. 일반 BFS나 Dijkstra와 비교했을 때 구현 조건이 더 단순한가?

## 대표 문제로 연결하기

### 문제에서 보이는 신호

- 입력 크기: 정점과 간선이 커서 `O(E log V)`도 부담될 수 있다.
- 필요한 복잡도: `O(V + E)`.
- 이 레슨의 핵심 개념: 비용 0은 현재 거리 그룹, 비용 1은 다음 거리 그룹으로 분리한다.

### 풀이 흐름

1. 문제의 상태를 정점으로 정의한다.
2. 각 전이를 비용 0 또는 1로 분류한다.
3. relax가 성공하면 비용 0은 deque 앞, 비용 1은 deque 뒤에 넣는다.

### 자주 틀리는 지점

- "방문한 정점은 다시 안 본다"는 BFS 습관을 그대로 쓰면 더 짧은 0비용 경로를 막을 수 있습니다.
- 비용 종류가 0/1을 벗어나면 Dijkstra로 바꿔야 합니다.
